Working with an AI assistant

NamDesktop exposes an MCP server that lets an AI assistant — such as Claude — read and update your workspace through natural conversation. You can ask it to capture inbox items, organise actions into projects, mark things done, or give you a summary of what is on your plate.

How it connects

NamDesktop starts an MCP stdio server automatically when the app launches. Configure your AI client (e.g. Claude Desktop) to connect via the MCP server entry for NamDesktop. Once connected, the assistant can see your inbox, projects, next actions, backlog, and any saved filters.

Monitoring mode — reads vs. writes

Reading is always available. Writes require monitoring mode to be active — this is a deliberate safety gate so the AI cannot modify your workspace without your knowledge.

Enable monitoring mode by pressing Cmd+Shift+M (macOS) or clicking the antenna icon in the toolbar. An amber ● Monitoring badge appears to confirm it is active. While monitoring mode is on, the assistant can create projects, add actions, move items, attach resources, and delete nodes.

Checkpoint and Accept

After the assistant has made changes, press Checkpoint (✔ in the toolbar, or Cmd+Shift+S) to review what it did. A summary dialog lists every addition, edit, and deletion. Accept to keep the changes; reject to discard them. Either way, monitoring mode stays active so you can continue the conversation.

If you close NamDesktop while monitoring mode is active, the same accept/reject dialog appears — no changes are silently lost.

What the assistant can do

  • List inbox items, next actions, backlog, projects, done items, and saved filters
  • Create projects and nest them under parent projects
  • Add inbox items, next actions, and project actions
  • Mark items as next, done, or backlog
  • Attach and remove resources (URIs, files, notes) on any node
  • Find nodes by title
  • Delete leaf nodes (nodes with no children)

Tips

  • Start a session by asking the assistant to summarise your inbox or next actions — it reads everything before suggesting changes.
  • Enable monitoring mode only when you are ready for the assistant to make changes. Leave it off for read-only conversations.
  • Use Checkpoint frequently during longer sessions so you can review changes in small batches.
