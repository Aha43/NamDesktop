Cloud sync

Cloud sync stores your workspace in a Supabase database so you can push it from one machine and pull it on another. It is the database-backed alternative to Git sync — no Git repository needed, and conflicts are detected instead of silently overwritten.

Setting it up

  1. Open Settings → Sync and fill in the Cloud sync (Supabase) group: the Supabase URL and publishable key (pre-filled for a local stack), and your account email and password.
  2. Tick Enable. The Push and Pull buttons in the toolbar now sync to Supabase.

Pushing and pulling

Click Push workspace (↑, or Cmd+S) to upload your workspace. Click Pull workspace (↓) to replace your local workspace with the remote one — the app reloads it immediately, no restart needed. The status bar confirms each sync with a timestamp.

Conflicts

If the remote workspace changed since your last sync (for example, you pushed from another machine), pushing opens a Workspace conflict dialog with three choices:

  • Keep remote — pulls the remote version and replaces this workspace. Local changes since your last sync are lost.
  • Keep local — overwrites the remote workspace with this one.
  • Cancel — does nothing, so you can look around before deciding.

Either choice is safe for a personal workspace — pick whichever copy is the one you want to keep going forward.

Git sync and cloud sync

Both can be configured, but the toolbar buttons follow cloud sync when it is enabled. Untick Enable in Settings to fall back to Git sync.

Dev mode

Cloud sync also works in dev mode — unlike Git sync — and targets a separate remote workspace named dev. Your real workspace and the dev sandbox sync independently and never mix: pushing from dev mode cannot touch your normal workspace, and vice versa. Tooltips and the status bar say (dev) when syncing the dev workspace.
