Column view

Column view shows a project's structure as a board. When a project has sub-projects, open its workbench and click the columns icon in the breadcrumb bar.

  • Each column is one immediate sub-project.
  • Each card is an action belonging directly to that sub-project.
  • A leading Unsorted column appears when the project has actions of its own that don't yet live in a sub-project, so you can sort them into one.

It answers "which part of this project does this action belong to?" at a glance — something the stacked workbench sections can't show side by side.

Moving and reordering cards

Drag a card onto another column to move it there, or drag it up or down within its column to reorder it. Prefer not to drag? Select a card and click the move icon at the bottom of its column (or right-click the card), then pick the destination column.

Moving a card to another column reparents the action to that sub-project — Column view never invents a separate board state; moving a card simply changes which sub-project owns the action (and where it sits in the order), exactly as if you had moved it in the workbench.

Click a column's title to drill into that sub-project. Use the columns icon again, or the readiness icon, to switch views; Column view is available whenever the project has at least one sub-project.

Lanes

With Column view open, the lanes button cycles each column through three layouts: actions only (the default), actions and sub-projects (two lanes), and sub-projects only. Drag an action or a sub-project to another column to reparent it; double-click a sub-project to drill into it.

Collapsing and rearranging columns

Each column header has a chevron to collapse it into a narrow strip, so the columns you care about right now get the room. Collapsed columns are remembered per project for the session. Use the ◀ ▶ buttons on a column to move it left or right — because a column is a sub-project, this reorders the sub-projects themselves, so the new order also shows in the Workbench view. The Unsorted column always stays first.

Switching views and getting back

The breadcrumb bar has three view buttons — Workbench (the default stacked view), Columns, and the readiness board — pick one to switch; pick Workbench to return to the plain view.

The view sticks

Each project remembers the view you left it in — Workbench, Columns (with whichever lanes), or the readiness board — so navigating away and back returns you to it. This lasts for the session; after a restart projects open in the normal workbench again.
