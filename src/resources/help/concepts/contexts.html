Contexts

A context is a condition that must be true for you to do an action. Classic GTD contexts are places or tools: @phone, @computer, @home, @errands.

In NamDesktop, contexts are implemented as tags. Tag an action or project with @home and it shows up when you filter by @home in the Context view.

Tags on a project are inherited by all actions inside it. So tagging a project @home automatically makes all its actions available in the @home context — without tagging each action individually.

The Context view uses AND logic: if you select both @home and @computer, you'll only see actions that match both tags.

You can save a tag combination as a Saved View so you can return to it quickly. For example, save @home + @computer as "Home office" for your home-working sessions.
