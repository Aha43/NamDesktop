Git sync

Git sync lets you back up and share your workspace across machines using a private GitHub repository. NamDesktop clones the repo locally, copies your workspace.json into it, and pushes — or pulls the latest version down when you switch machines.

Setting it up

  1. Create a private GitHub repository for your workspace (no README needed).
  2. Make sure git is installed and authenticated on your machine (e.g. via SSH key or GitHub CLI).
  3. Open Settings in NamDesktop and paste the repo URL into Sync repo URL (e.g. https://github.com/you/my-workspace).
  4. Save Settings. Push and Pull buttons appear in the toolbar and under File in the menu bar.

Pushing and pulling

Click Push workspace (↑) to save your current workspace to the remote repo. Click Pull workspace (↓) to overwrite your local workspace with the version from the remote. NamDesktop saves before pushing and reloads after pulling.

Pull replaces your local workspace. Make sure you have pushed from your other machine first, or you will overwrite unsaved work.

How it works

On first use NamDesktop clones the repo into a local cache folder. Each push copies workspace.json into the clone, commits with the message sync, and runs git push. Each pull runs git pull and copies the updated file back into your workspace location. No history is kept beyond what Git stores in the repo.
