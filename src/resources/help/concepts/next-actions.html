Next Actions

A next action is the next physical step you can take to move something forward. "Organise the garage" is not a next action — it's a project. "Buy storage boxes" is a next action.

The test: could you sit down right now and do it? If yes, it's a next action. If it needs something else first, break it down further.

If you know GTD: this maps directly to your next-actions list — the committed, immediately doable steps.

In NamDesktop, Next Actions are all the tasks currently marked with Next status. They appear in the Next Actions view grouped and ordered. You can drag them up and down to set your own priority.

Each action shows its full project path so you always know where it belongs. The path is a breadcrumb of links — click any segment to jump straight to that project. This works wherever an action lists its project path: Next Actions, Backlog, Context, Due, Blocked, Done, saved views, and the focus deck.

When you complete an action, right-click → Mark done. It moves to the Done list where you can review it later.

Bulk actions — tick the checkbox at the end of one or more rows (or the header checkbox to select all) and a bar appears with Set status (Next / Backlog / Done), Add tag, and Delete, applied to every checked action at once. The same bulk bar is available in Backlog, Done, Context, Saved Views, Due, and Blocked — and in the Project Workbench, where ticks are shared across a project's own actions and its sub-projects, so you can tidy up a whole project at once.

If an action isn't ready yet, move it to the Backlog so it doesn't clutter your active list.

If an action is waiting on another action to finish first, mark it as blocked — it will reappear automatically once its blocker is done.

Age column

Each row shows an Age value — how long since the action was last touched (3d, 2w, 4m…). A next action that hasn't been looked at in weeks may not really be "next" anymore — use it as a prompt to reconsider or move it to the Backlog.

Due column

If an action has a due date set, a Due column shows how close the deadline is: overdue in red, today in amber, this week as a day name, later as a short date. Open any action (double-click or pencil icon) to set or clear its due date.

Sort order

The Sort button cycles through three states:

  • Sort (inactive) — your manual priority order is respected
  • Oldest — oldest actions first; surfaces what has been waiting longest
  • Newest — most recent first

When a sort is active, the up/down ordering buttons are hidden to avoid conflict. The sort preference is saved across restarts.
