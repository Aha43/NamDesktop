Projects

A project is anything that takes more than one step to finish. "Write report" is a project. "Send the report" is a step.

Projects don't have to be big. A dentist appointment is a project if it requires: find a dentist, call to book, attend appointment.

If you know GTD: this is the same definition — any desired outcome requiring more than one action.

In NamDesktop, a project node holds actions and sub-projects inside it. Open any project to enter the Project Workbench, where you can see all actions and sub-projects at once.

The Projects view has three layouts, switched from the toolbar: the compact List (default), a Column view laying each top-level project out as a column of its actions, and a Readiness view showing a progress heat-map card per project. The Filter by tag strip narrows all three; the layout you choose is remembered across restarts.

Tags on a project are inherited by all actions inside it. This means you can tag a project @home and all its actions will show up when you filter by @home in the Tag filter view.

Sub-projects let you break a large goal into phases. Each sub-project has its own actions and can be tracked individually.

You can attach resources — links, file paths, or notes — to any project or action to keep reference material close to the work.

If you have a project structure worth repeating, save it as a template and apply it to future projects.
