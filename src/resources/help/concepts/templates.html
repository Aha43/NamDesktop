Project Templates

A template is a saved project structure you can stamp out as many times as you need. Once you have a pattern that works — a trip checklist, a sprint plan, a weekly review — save it as a template and apply it to any new project with one click.

Creating a template

Open a project that already has the structure you want to reuse — its sub-projects and actions become the template. In the project dialog (click the pencil icon), click Save as Template… and give it a name. The entire sub-project and action hierarchy is captured as-is.

Applying a template

Open File → Templates… to see all saved templates. Select the template you want, then click Apply to project… and pick a target project. The template's structure is added as new child sub-projects and actions inside that project. Existing content is left untouched.

When templates are useful

  • Repeating project patterns — a sprint, a trip, an event setup
  • Checklists with sub-structure — a weekly review with phases, a product launch with workstreams
  • Onboarding new goals — stamp out a proven structure for any new project of the same type

Managing templates

Open File → Templates… to view and delete saved templates. Templates are stored in your workspace file, so they sync with Git sync automatically.
