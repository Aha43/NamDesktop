Project Workbench

The Project Workbench is where you manage a single project in depth. Open it by clicking any project in the Projects list or by clicking a project path link in any action view.

The workbench shows:

  • A breadcrumb bar at the top — click any ancestor to navigate up
  • Sub-projects — each as a collapsible section with its own action list
  • This project — actions that belong directly to the current project

From the workbench you can:

  • Add actions and sub-projects
  • Quick-rename or edit descriptions inline
  • Delete sub-projects (with a warning showing how many items will be removed)
  • Navigate into any sub-project by clicking its name
  • Save the project's structure as a template to reuse on future projects
  • Attach resources (links, files, notes) to the project or any of its actions
  • Bulk actions — each action row has a checkbox; tick actions (across the project's own list and any sub-project) and a bar appears with Set status, Add tag, and Delete applied to all checked actions at once — handy for tidying up a project.
  • Copy a Markdown summary — the copy button in the breadcrumb bar opens a summary dialog: a live preview of the project as a Markdown checklist (actions as - [ ] / - [x], sub-projects as nested headings), an Include sub-projects toggle, and a Copy to clipboard button. Handy for pasting a status snapshot into a chat or an AI assistant.

Readiness view

If your project has sub-projects, the breadcrumb bar shows a dashboard icon (⊞). Click it to switch to readiness view — a heat-map of sub-projects showing how much work is done in each one at a glance. Click any card to drill into that sub-project; use Back to return to the readiness overview.

Column view

The columns icon next to it switches to Column view — each sub-project becomes a column and you can move actions between them.
